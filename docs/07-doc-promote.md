# Delivery. Doc-promote — поведение в канон

![Два канона: ADR-promote и doc-promote](assets/canons_adr_promote_docs_promote.png)

```
/spec-ship:doc-promote
```

## Что это

Переносит знание о **поведении системы** из одноразовых артефактов фичи (survey, BusinessDoc) в долгоживущие workflow-доки `.ship/docs/workflows/` с индексом и жизненным циклом.

Зеркало adr-promote, но для другого вида знания:

| | ADR-канон | Workflow-канон |
|---|---|---|
| Хранит | «почему решили так» | «как система себя ведёт» |
| Источник | кандидаты из build | survey + BusinessDoc после мёржа |
| Главный читатель | shape-doc, decompose, build, review | survey следующей фичи |

Это тоже **ручной шаг maintainer'а** после мёржа MR — канон описывает реальность в ветке, а не намерения.

## Зачем

Без него каждый survey по одной и той же области трассирует код заново — это дорого. С каноном survey сначала читает готовый док (только актуальные, только по нужной области через индекс) и трассирует лишь расхождения и непокрытые код-пути. Каждая следующая фича в области дешевле предыдущей.

## Как проходит

1. **Предусловия:** ревью APPROVED, MR смёржен.
2. **Целевой документ.** Область выводится из затронутых файлов. Если док по этому код-пути уже есть — режим amend (обновить), нет — создать.
3. **Сборка.** Итоговый документ описывает состояние ПОСЛЕ мёржа: workflow из BusinessDoc замещают изменённые ими наблюдения survey, незатронутые наблюдения переносятся как есть. Если `feature.workflow` записан рефакторинг-формой `{старый} --шаг--> {новый}` — левая часть указывает, какой workflow канона замещается, в документ попадает правая. Стрелочные строки копируются из артефактов, не пересочиняются. Формат дока: Definitions (структуры, таблицы, события), Workflows (стрелочная нотация + пояснения), словарь имён состояний. Документ читается без открытия кода.
4. **Индекс.** Строка в `.ship/docs/workflows/INDEX.md`: документ, статус, область, источник, дата. Если фича задела поведение, описанное в **других** доках, — им проставляется честный статус: `partially-outdated` или `superseded`. Молча лгущий документ — худший исход, статусы обязательны.

## Жизненный цикл документов

| Статус | Значение |
|---|---|
| `current` | авторитетно описывает поведение в ветке; survey ему доверяет |
| `partially-outdated` | частично устарел; что именно — отмечено внутри; кандидат на amend |
| `superseded` | замещён другим документом; только история |

## Что от вас потребуется

- Инициировать после мёржа
- Подтвердить, какие чужие доки получили `partially-outdated`/`superseded`
